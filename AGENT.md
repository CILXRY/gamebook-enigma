# Project Intro (项目简介)

这是一个用 Flutter 开发的跨平台“游戏本子”App。
核心功能是帮助玩家记录不同游戏的进度、账号、区服和备注信息。
当前项目处于早期 MVP 阶段，主打快速迭代、尽可能高可用性的哲学。

# Tech Stack (技术栈)

- **框架**: Flutter (Dart)
- **状态管理**: 暂定 Provider (或 setState)，拒绝过度工程化
- **本地存储**: shared_preferences
- **样式**：遵循 Material Design 3 风格

# 🎨 Coding Style (代码风格)

- 严格遵循 Dart 官方 Effective Dart 规范。
- 变量和函数命名要有意义，拒绝 `a`, `b`, `test1` 这种命名。
- 尽量写点注释，特别是在业务逻辑比较复杂的地方。

# 🤝 AI Collaboration Rules (与 AI 协作规则)

1. **让我们说中文**：没别的意思，中文交流而已
2. **解释优先**: 在给出复杂代码前，先用大白话解释实现思路。
3. **新手教学**: 遇到 Flutter 特有概念（如 Widget 生命周期、异步处理），请顺带科普一下。
4. **极简主义**: 不要引入不必要的第三方库，能原生实现就用原生。
5. **报错处理**: 如果代码报错，不要盲目试错，请分析可能的原因并给出排查建议。
6. **无需编译**：在编写代码过程中我会自己来调试，不需要你来编译测试。
7. **低耦合设计**：在设计的时候将UI和业务逻辑拆开来写
